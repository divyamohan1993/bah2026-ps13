# NETRA — FULL dependency manifest (all tiers)
# ============================================
# Organised into tiers by weight. The CORE tier is the contract for the
# CPU-only / no-GPU / no-internet demo and is duplicated (verbatim pins) in
# requirements-core.txt — keep the two in sync. The OPTIONAL-HEAVY tiers add
# deep models, foundation forecasters, the offline 7B LLM and the RAG stack;
# they materially improve quality but the system DEGRADES GRACEFULLY without
# them (deep models skipped, copilot falls back to a deterministic template that
# emits the same CopilotResponse schema).
#
# AIR-GAP RULE: nothing here calls a cloud API at runtime. `nixtla` (TimeGPT) is
# deliberately EXCLUDED — it is a cloud service and must never enter the runtime
# path. HF weights for the heavy tiers are pre-downloaded once and loaded by
# local path with HF_HUB_OFFLINE=1 / TRANSFORMERS_OFFLINE=1.
#
# Offline install (full):
#   pip download -r requirements.txt -d wheelhouse        # connected host
#   pip install --no-index --find-links=wheelhouse -r requirements.txt
# Per-module extra deps may live in netra/<module>/requirements-<module>.txt and
# are folded in by the integrator.
